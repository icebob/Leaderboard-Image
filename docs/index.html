Áttekintés

A Képgenerátor Aréna egy web-alapú alkalmazás, amely lehetővé teszi különböző AI képgenerátorok által létrehozott képek összehasonlítását és értékelését. A rendszer három fő módot kínál:

  • Arena Battle: Két kép közvetlen összehasonlítása, ahol a felhasználók a jobbnak ítélt képre szavazhatnak.
  • Side-by-Side: Két kiválasztott modell képeinek összehasonlítása egymás mellett.
  • Leaderboard: A modellek ranglistája az ELO pontszámok és egyéb statisztikák alapján.
Jelenleg támogatott modellek
  • Grok
  • Google Gemini Flash
  • Google Imagen 3
  • ChatGPT GPT 4o
  • Midjourney
  • Reve

Funkciók

ELO Rating

Fejlett ELO pontrendszer, amely figyelembe veszi az ellenfelek erősségét a modellek rangsorolásában.

Több formátum támogatása

A rendszer támogatja a JPG, JPEG és PNG formátumú képeket.

Konfigurálhatóság

A modellek, fájlformátumok és alapbeállítások külön konfigurációs fájlban vannak tárolva.

Vak szavazás

Arena Battle módban a modellek nevei csak a szavazás után jelennek meg.

Összehasonlítási módok

Többféle mód a képek összehasonlítására és értékelésére.

Részletes statisztikák

ELO pontszámok, győzelmek, mérkőzések száma és győzelmi arányok nyomon követése.

Üzemmódok

Arena Battle

Az Arena Battle a rendszer fő módja, ahol két véletlenszerűen kiválasztott modell által generált kép jelenik meg egymás mellett. A felhasználók kiválaszthatják, melyik kép tetszik jobban, vagy döntetlen/kihagyás opciót választhatnak.

Különleges funkció: A modellek nevei csak a szavazás után jelennek meg, így biztosítva az elfogulatlan értékelést.

  • A szavazás után a rendszer frissíti az ELO értékeket
  • A modellek nevei 1,5 másodpercre láthatóvá válnak a szavazás után
  • Automatikusan betöltődik a következő összecsapás

Side-by-Side

A Side-by-Side módban a felhasználók maguk választhatják ki, melyik két modellt szeretnék összehasonlítani. Ez a mód elsősorban vizuális összehasonlításra szolgál, nincs szavazás.

Ez a mód hasznos lehet:

  • Specifikus modellek közvetlen összehasonlítására
  • A modellek erősségeinek és gyengeségeinek alaposabb vizsgálatára
  • Oktatási célokra, hogy bemutassa a különböző modellek képességeit

Leaderboard

A Leaderboard a modellek ranglistáját mutatja ELO pontszám szerint csökkenő sorrendben. A táblázat tartalmazza az ELO értékeket, a győzelmek számát, az összes mérkőzés számát és a győzelmi arányt.

ELO rendszer: A rangsorolás a sakk-világból ismert ELO pontrendszeren alapul, amely figyelembe veszi az ellenfelek erősségét.

  • Az új modellek alapértelmezett ELO értékkel (1500) kezdenek
  • A szavazások hatására a pontok dinamikusan változnak
  • Az erősebb ellenfél legyőzéséért több pont jár

Telepítés és indítás

Követelmények

  • Python 3.6+
  • pip (Python csomagkezelő)
  • Git (opcionális)

Telepítési lépések

# 1. Klónozd vagy töltsd le a repository-t
git clone https://github.com/yourusername/image-leaderboard.git
cd image-leaderboard

# 2. Függőségek telepítése
pip install -r requirements.txt

# 3. Adatbázis inicializálása
python database.py

# 4. Alkalmazás indítása
flask run --host=0.0.0.0

# Az alkalmazás alapértelmezetten a következő címen érhető el:
# http://localhost:5000

Parancsok magyarázata

python database.py

Ez a parancs inicializálja az adatbázist. Létrehozza a szükséges táblákat (votes és model_elo) az SQLite adatbázisban, ha még nem léteznek, és beállítja az alapértelmezett ELO értékeket a modellekhez. A parancs futtatása után ellenőrzi a data mappában található prompt ID-kat is.

flask run --host=0.0.0.0

Ez a parancs elindítja a Flask webszervert. A --host=0.0.0.0 paraméter lehetővé teszi, hogy az alkalmazás ne csak a localhost-ról, hanem a hálózat más eszközeiről is elérhető legyen. Ez különösen hasznos, ha más eszközökről (pl. mobiltelefon, tablet vagy más számítógép) is szeretnéd elérni az alkalmazást ugyanazon a hálózaton belül.

python app.py reset-votes

Ez a parancs törli az összes eddigi szavazatot és visszaállítja az ELO pontszámokat az alapértelmezett értékre. Ezt akkor érdemes használni, ha:

  • Teljesen új versenyt akarsz indítani
  • Tesztadatok után szeretnéd az éles adatgyűjtést elkezdeni
  • Problémás szavazatok kerültek a rendszerbe

Konfigurálás

A rendszer konfigurációs beállításai a config.py fájlban találhatók. Itt módosíthatók a következő beállítások: 

# Adatbázis fájl neve
DATABASE = 'votes.db'

# Az adatmappák elérési útja
DATA_DIR = 'data'

# Modell nevek és a hozzájuk tartozó fájlkiterjesztések/nevek
MODELS = {
    'grok': 'grok.jpg',
    'gemini-flash': 'gemini-flash.jpeg',
    'gemini-25pro': 'gemini-25pro.jpg',
    'gpt4o': 'gpt4o.png',
    'midjourney': 'midjourney.png'
}

# Engedélyezett képkiterjesztések listája
ALLOWED_EXTENSIONS = ['.jpg', '.jpeg', '.png']

# ELO Rating beállítások
DEFAULT_ELO = 1500  # Alapértelmezett ELO érték új modellekhez
K_FACTOR = 32       # K-faktor - a magasabb érték nagyobb változást eredményez

Új modell hozzáadása

Új modell hozzáadásához a következő lépéseket kell követni:

  1. Helyezd el az új modell által generált képeket minden prompt mappában a megfelelő fájlnévkonvencióval
  2. Add hozzá a modellt a config.py fájl MODELS szótárához
  3. Indítsd újra az alkalmazást, hogy az új modell megjelenjen a felületeken

Rugalmas fájl kiterjesztések

A rendszer képes rugalmasan kezelni a képfájlok kiterjesztéseit. Ez azt jelenti, hogy:

  • Ugyanazon modell képei különböző kiterjesztésekkel szerepelhetnek különböző prompt mappákban
  • Támogatott kiterjesztések: .jpg .jpeg .png
  • A fájlnév alaprésze (kiterjesztés nélkül) mindig meg kell hogy egyezzen a konfigurációban beállítottal
Példa: Ha a config.py fájlban a 'Grok': 'grok' beállítás szerepel, akkor a rendszer automatikusan megtalálja a grok.jpg, grok.jpeg vagy grok.png fájlokat a prompt mappákban. 
# Modell nevek és a hozzájuk tartozó fájl alapnevek (kiterjesztés nélkül)
# Fontos: Ezek csak a fájlok alapnevei, a kiterjesztés dinamikusan lesz meghatározva
MODELS = {
    'Grok': 'grok',
    'Google Gemini Flash': 'gemini-flash',
    'Google Imagen 3': 'imagen3', 
    'ChatGPT GPT 4o': 'gpt4o',
    'Midjourney': 'midjourney',
    'Reve': 'reve'
}

Hibaelhárítás

Képek nem jelennek meg

Ha a képek nem jelennek meg:

  1. Ellenőrizd, hogy a fájlnevek megfelelnek-e a konfigurációban beállítottaknak
  2. Ellenőrizd, hogy a kiterjesztés a támogatottak között van-e (.jpg, .jpeg, .png)
  3. Ellenőrizd a böngésző konzolját hálózati hibákért
Adatbázis hibák

Adatbázis problémák esetén:

  1. Próbáld meg törölni a votes.db fájlt és futtasd újra a python database.py parancsot
  2. Használd a python app.py reset-votes parancsot a szavazatok visszaállításához
  3. Ellenőrizd a terminál kimenetét a hibaüzenetekért
Tipp: Az alkalmazás debug módban további információkat jelenít meg a konzolon. Ezek az információk segíthetnek a hibák azonosításában.
